## Data Schemas
## Data Schemas

### 1. `User` Schema
The authenticated user who owns notes and sources.
```json
{
  "id": "uuid",
  "email": "string",
  "created_at": "timestamp"
}
```

### 2. `Source` Schema
The raw materials uploaded by the user (transcripts, textbook excerpts).
```json
{
  "id": "uuid",
  "user_id": "uuid",
  "title": "string",
  "content": "text",
  "type": "enum: [transcript, textbook, handwriting_ocr, web_clip]",
  "created_at": "timestamp"
}
```

### 3. `Note` Schema
The professional-level output generated by AI or typed by the user.
```json
{
  "id": "uuid",
  "user_id": "uuid",
  "title": "string",
  "content": "text (markdown supported)",
  "source_ids": ["uuid"],
  "is_ai_generated": "boolean",
  "last_autosaved": "timestamp",
  "created_at": "timestamp"
}
```

---

## Behavioral Rules
1. **User Segregation:** A user may only access their own `User`, `Source`, and `Note` records (enforced via RLS).
2. **Auto-Save:** Any edits to a `Note` trigger an auto-save (debounced to avoid spamming the DB).
3. **Markdown Support:** All `Note` contents are rendered as markdown on the frontend.
4. **AI Generation:** When AI generates a note, it strictly references the provided `Source` materials and tags `is_ai_generated=true`.

---

## Architectural Invariants
- **3-Layer Architecture:** `architecture/` (SOPs) → Navigation (routing) → `tools/` (deterministic scripts)
- **Data-First:** No coding begins until payload shapes are confirmed
- **Deterministic Logic:** Business logic lives in `tools/`, never in LLM prompts
- **Self-Annealing:** All errors are analyzed, patched, tested, and documented
- **Intermediates in `.tmp/`:** Ephemeral files only; payloads go to final destination
